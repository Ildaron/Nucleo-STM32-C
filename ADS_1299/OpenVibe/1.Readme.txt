1. Драйвер должен объявить свое имя серверу сбора данных.
2. Драйвер имеет дело с двумя видами данных: заголовок (the header);
                                             буфер (the buffer)
Заголовок - это часть данных, которая не изменяется со временем. Он содержит несколько идентификаторов о проводимом эксперименте, информацию о приобретаемых каналах.
Буфер - это часть данных, которая меняется со временем. Он содержит различные выборки для каждого канала в данный период времени, которые зависят от количества выборок на отправленный канал
3. Драйвер вызывается сервером:
   - конфигурация;
   - инициализация/деинициализация;
   - получение.
4. In the initialization stage, the driver requests readiness status from the hardware. 
   The driver is initialized when OpenViBEAcquisitionServer::IDriver::initialize returned   
5. Цикл драйвера OpenViBEAcquisitionServer :: IDriver :: регулярно вызывается, чтобы драйвер мог поддерживать соединение с устройством живым, отбрасывая при необходимости некоторые данные.
6. Этап сбора данных происходит, когда вызывается OpenViBEAcquisitionServer :: IDriver :: start . На этом этапе драйвер регулярно запрашивается для предоставления новых данных в функции OpenViBEAcquisitionServer :: IDriver :: loop .
7.Чтобы драйвер был виден на сервере сбора данных, вам необходимо зарегистрировать драйвер. Это происходит следующим образом,

Поместите ваши файлы .h / .cpp в папку contrib / plugins / server-drivers / your-driver-name / src / .
Отредактируйте contrib / common / contribAcquisitionServer.inl . Этот встроенный файл компилируется с Acquisition Server и регистрирует драйвер.
Отредактируйте contrib / common / contribAcquisitionServer.cmake . Этот скрипт должен сделать ваш драйвер видимым для сборки.
Отредактируйте contrib / common / contribAcquisitionServerLinkLibs.cmake . Этот скрипт должен вызывать скрипт поиска CMake, связанный с вашим драйвером. Вызываемый скрипт должен проверить, присутствуют ли в системе компоненты (если таковые имеются), требуемые вашим драйвером, и добавить их в сборку, если это так. Файл cmake должен сделать все заголовки и необходимые библиотеки видимыми для компиляции и установить все необходимые файлы времени выполнения (например, .dll) в каталог dist / bin /. Пожалуйста, используйте include guard в вашем драйвере, чтобы сборка не пыталась скомпилировать ваш драйвер, если его зависимости недоступны.
8. Запуск на linux из папки dist/extras-Release
